Interactive Degree Planner

Drag-and-drop course planning with live designation gap analysis

Published

May 28, 2026

TipThe planner is live

Open the interactive planner in a new tab: obwb.shinyapps.io/ubco-freshwater-planner/

The app reads the live UBCO course catalogue and the PAg + RPBio designation mappings curated on this site. Tick the courses you’ve completed (or transferred in), pick one or both designation goals, and the app reports:

  • credits + degree-threshold check (120 / 78 Science / 42 upper / 36 upper Science)
  • PAg category coverage (8 foundational + 20 agrology with at least 8 senior)
  • RPBio category coverage (6 core areas + 5 three-of-five areas) plus the 13-biology count

This is v1: it’s intentionally simple (no drag-drop year slots yet, no prereq DAG viz). v2 will add those features once the v1 has been in front of students for a term.

What it will do

When live, the planner will let you:

  • Drag courses into year slots (Year 1, 2, 3, 4 and post-Year-4 summer).
  • Toggle designation goals (PAg, RPBio, neither, both).
  • See live gap analysis:
    • Which prerequisites are missing for the courses you have chosen.
    • Which BCIA / CAB category requirements are still unfilled.
    • Whether your total credits clear the 120 / 78 Science / 42 upper-level / 36 upper-level-Science thresholds.
  • View the prerequisite graph for your selected courses (visNetwork).
  • Export your plan as a PDF or text summary you can share with the program advisor.

In the meantime

  • Use the course catalogue for the year-by-year course list and prerequisite notes.
  • Use the PAg pathway and RPBio pathway for the curated 4-year sample plans + designation-fit narrative.
  • Use the Complementary pathways page for Chemistry, Biology, and hydrology/water-resources bundles.

Want to help shape it?

If you are a Freshwater Science student or instructor and want input into how the planner should behave, what gap signals matter most, what visual encoding is most useful, how aggressively it should flag prerequisite conflicts, please drop a note to Nelson.Jatel@UBC.ca or open a GitHub issue at the project source repository.
